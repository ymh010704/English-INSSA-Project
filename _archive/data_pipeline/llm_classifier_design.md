# LLM 슬랭 분류기 설계

## 1. 핵심 근거

**match_count가 높을수록 일반어/다의어로 쓰이는 비율이 높아진다.**

Wiktionary에서 slang 레이블로 추출된 단어라도, Reddit에서 match_count가 높다는 것은
그 단어가 slang 외의 일반적 의미로도 많이 쓰인다는 직접적 증거다.
따라서 match_count ↑ → 슬랭 비율 p ↓ 가 성립한다.

예: "fire" (match=1.7M) → 대부분 "불/해고" 의미, 슬랭("awesome")은 일부
    "rizz" (match=21K)  → 거의 슬랭으로만 사용


## 2. Tier별 슬랭 비율 p 및 수집량 n

p는 match_count 구간별로 감소하게 설정.

**n 결정 기준:** P(슬랭 문맥 0건 | p, n) < 0.5%
```
(1-p)^n < 0.005
```
이 부등식을 만족하는 최소 n은 tier별로 33, 51, 104, 174, 263개.
실제 수집량은 운용 편의를 위해 올림한 round number 사용 (50, 100, 150, 200, 300).
모두 P < 0.5% 기준을 충족함을 검증.

수집 단계(reddit_context_cache_builder.py)에서 이미 완료.

| Tier | match_count | 단어 수 | p 가정 | n 최소 | n 실제 | P(슬랭 0건) |
|------|------------|--------|-------|--------|--------|------------|
| 0 | <10K | 667 | 15% | 33 | 50 | 0.03% |
| 1 | 10K~50K | 2,614 | 10% | 51 | 100 | 0.003% |
| 2 | 50K~200K | 1,534 | 5% | 104 | 150 | 0.046% |
| 3 | 200K~1M | 1,056 | 3% | 174 | 200 | 0.24% |
| 4 | >1M | 552 | 2% | 263 | 300 | 0.23% |


## 3. 초기 배치 크기 n_init

901,650개 문맥 전부를 LLM에 전송하면 비용이 과도하다.
초기에 소량만 전송해서 슬랭 비율이 0이면 즉시 DROP → 비용 절감.

**n_init 도출 기준:** P(슬랭 0건 발견 | 실제 슬랭 단어) < 5%
```
(1-p)^n_init < 0.05
n_init = ceil(log(0.05) / log(1-p))
```

| Tier | p | n_init | n_collected | DROP 시 절감 |
|------|---|--------|------------|------------|
| 0 | 15% | 19 | 50 | 31개 |
| 1 | 10% | 29 | 100 | 71개 |
| 2 | 5% | 59 | 150 | 91개 |
| 3 | 3% | 99 | 200 | 101개 |
| 4 | 2% | 149 | 300 | 151개 |

초기 배치 합계: 365,777개 (전체 901,650개의 40.6%)


## 4. Adaptive 분류 전략 (ratio 기반)

단순 카운트가 아닌 **비율(ratio)**로 판단 → tier별 배치 크기 차이를 자연 흡수.

```
ratio = 누적 slang_count / 누적 n_sent

[매 배치 전송 후 판단]
ratio == 0              → DROP (즉시 종료)
ratio >= p/2            → KEEP 확정 (종료)
0 < ratio < p/2         → 증거 불충분, 추가 배치 전송
n_sent == n_collected   → 문맥 소진, 최종 판단:
    ratio >= p/3        → KEEP
    ratio < p/3         → DROP
```

**임계값 근거:**

- `p/2`: 초기 배치(n_init)에서 ratio >= p/2이면 slang_count ≈ **1.5개** 이상.
  모든 tier에서 동일하게 수렴 → 초기 배치에서 슬랭이 ~2건 이상 관측되면
  추가 확인 없이 KEEP 판단할 수 있는 최소 evidence로 설정.

- `p/3`: 전체 문맥(n_collected) 소진 후 ratio >= p/3이면 slang_count ≈ **2~3개** 이상.
  모든 tier에서 동일하게 수렴 → 학습 데이터 항목으로 포함하기 위한
  최소 슬랭 문맥 수(2~3개)를 ratio 조건으로 표현한 것.
  문맥을 더 이상 추가할 수 없는 상태이므로 기준을 p/2에서 p/3으로 완화.

**배치 크기 전략 (미확정):**
- 초기: n_init
- 이후: n_init씩 추가 OR 나머지 전부 한 번에
- 논의 필요

## 6. KEEP 확정 후 처리 (미설계)

ratio >= p/3 기준을 통과한 단어는 전체 수집 문맥을 대상으로 sense 분류 진행.
이 시점에서 슬랭 문맥은 이미 최소 2~3개 이상 확보된 상태.

- 각 문맥 → 슬랭 sense A / 슬랭 sense B / 일반 의미 / 불분명 분류
- top sense 선정: 논의 필요
- 출력: word_summary.jsonl
